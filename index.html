Future模式浅析

发表于 2017-05-26 | 分类于 Java

Future模式

Future模式的核心是异步调用。有时候调用要很久才能返回，但是我们并不急于想要结果。因此我们可以让被调用者立即返回，让它在后台慢慢处理这个请求。对于调用者而言，则可以先处理一些其他任务，在真正需要数据的场合再去尝试获得需要的数据。

对于Future模式而言，虽然它无法立即给出想要的结果，但是，他会立即返回给你一个契约，将来，凭着这个契约可以取重新获取你想要的信息。
