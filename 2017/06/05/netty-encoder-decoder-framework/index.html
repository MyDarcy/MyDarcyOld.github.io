netty编码解码框架

发表于 2017-06-05 | 分类于 Netty | 阅读次数
| 字数统计

编解码器：数据从一种特定协议的格式到另一种特定格式的协议的转换。

编解码器

  • 编解码器：定义如何解析在两个节点之间来回传输的原始字节，以及如何将其和目标应用程序的数据格式做相互转换。
  • 编码器：消息转换为适合于传输的格式（一般是字节流）。处理出站数据。
  • 解码器：网络字节流转换回应用程序的的消息格式。处理入站数据

解码器

  • 字节解码为消息：ByteToMessageDecoder, ReplayingDecoder
  • 一种消息解码为另一种消息：MessageToMessageDecoder
  • 应用场景：每当需要为ChannelPipeline中的下一个ChannelInboundHandler转换入站数据时会用到。在ChannelPipeline中，可以将多个解码器链接在一起，以实现任意复杂的转换逻辑。

抽象类ByteToMessageDecoder

  • decode : 不断解码传入的bytebuf, 并且将解码后的消息添加到list中，直到bytebuf中没有更多可读取的字节或者list中没有再添加新的元素为止。然后如果list不为空，那么list中的内容被传递给ChannelPipeline中的下一个ChannelInboundHandler。
  • decodeLast : Channel的状态变为in-active的时候，调用本方法一次。

编解码器中的引用计数

  • 在编解码器中，一旦消息被编码或者解码，它就会被ReferenceCountUtil.release(msg)调用自动释放。如果需要将消息保留引用以便稍后使用，可以调用ReferenceCountUtil.retain(msg)方法，以增加引用计数，从而防止消息被自动释放。

抽象类ReplayingCodeder

  • ReplayingDecoder拓展了ByteToMessageDecoder类，其使用了一个自定义的ReplayingDecoderByteBuf类来包装ByteBuf，所以就不需要再显式调用readableBytes(), 而是在内部执行该调用。
  • 但是ReplayingDecoderByteBuf有如下缺陷：
    • 不是ByteBuf的所有操作都支持。
    • ReplayingDecoder比ByteToMessageDecoder慢一点。
