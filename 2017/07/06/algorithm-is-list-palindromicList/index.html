链表是否是回文

发表于 2017-07-06 | 分类于 刷题 | 阅读次数
| 字数统计

请编写一个函数，检查链表是否为回文。给定一个链表ListNode* pHead，请返回一个bool，代表链表是否为回文。这里有三种算法的实现，当作是拓展一下思路。

解法1–利用一个栈存储半个list.

  • 链表的前半部分入栈，根据链表长度已知与否，入栈有两种方式：a. 如果链表长度已知,那么for遍历半个list即可; b. 如果链表长度未知,那么fast-slow pointer, 将slow所指向的节点入栈.
  • 然后链表继续往下遍历，同时数据出栈，进行比较．

解法2– 递归法

  • 问题是要怎么才能比较节点0和最后一个节点n, 节点1和倒数第二个节点n-1;等等．这就需要先直到什么时候到达中间元素，即形成递归的终止条件．
  • 每次调用都会比较首部和returnNode, 然后回传returnNode.next．如此就能够一直让当前head和返回节点进行比较．直到得到最终结果．只要有任意一对节点的值不相等，立即返回false．调用栈的上一级调用都会检查这个布尔值.但是要同时返回下一个节点和当前值对是否匹配的bool值．所以就需要构造一个复合结构．见代码．具体的调用栈视图也在代码注释之中．

解法3– 反转链表

  • 这里要注意的是，反转链表的时候，要先构造原链表的副本．因为反转算法是在原链表上进行的．具体的实现比较简单，就不细说了．
