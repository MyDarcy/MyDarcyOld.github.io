简单记录一下Java8函数式编程的一些特点

函数式编程特点

函数式编程的特点和差异

函数作为一等公民

  • 函数可以作为另一个函数的参数和返回值

无副作用

  • 函数的副作用是指函数在调用过程中，除了给出返回值之外，还修改了函数外部的状态。比如修改了全局变量的值。

声明式的(Delarative)

  • 函数式编程是声明式的编程方式；命令式的编程方式涉及大量的可变对象和指令。而声明式编程不再需要提供明确的指令操作，所有的细节会被程序库所封装，你所要做的就是声明用意即可。

不变的对象

  • 函数式编程中，几乎所有传递的对象都不会被轻易的修改。

易于并行

  • 对象的不变性，易于并行，而且没有线程安全的问题。

函数式编程基础

FunctionalInterface注解

  • Java8提出了函数式接口，就是只定义了单一抽象方法的接口。但是函数式接口只有一个抽象方法，并不代表只能有一个方法：首先，Java8中，接口中可以定义默方法，其次任何被java.lang.Object实现的方法都不能视为抽象方法。
  • 注释FunctionalInterface用于表明IntHandler是一个函数式接口。但是如果一个函数满足函数式接口的定义，那么即使不标注为@FunctionalInterface，编译器依然会把它看作是函数式接口。类似于Override注解，主要起的作用是：如果进行了标注，但是实际代码不符合规范，那么机会得到一个编译期错误。
  • 函数式接口的定义可以由方法引用或者lambda表达式进行构造。

接口默认方法

  • 接口中可以定义默认方法的实现，如果实现的多个接口都声明了同一个方法，那么可以使用InterfaceName.super().method()让编译器进行方法绑定，相当于将特定的方法委托给父类实现。
  • Java8中，java.util.Comparator接口中新增了若干个默认的方法, 用于多个比较器的整合。

lambda表达式

  • lambda表达式即是匿名函数，是一段没有函数名的函数体，可以作为参数直接传递给调用者。
  • lambda表达式也可以访问外部的局部变量, 但是外部的局部变量必须被声明为final的或者隐式final的。

方法引用

  • 方法引用是Java8提出的简化lambda表达式的手段，可以通过类名和方法名来定位到一个静态方法或者实例方法
  • 方法引用分为如下几种
    • 静态方法引用: ClassName::methodName
    • 实例方法引用: instanceRef::methodName
    • 超类上的实例方法引用: super::methodName
    • 类型上的实例方法引用: ClassName::methodName
    • 构造方法引用: ClassName::new
    • 数组构造方法引用: TypeName[]::new
  • 流中的对象作为方法引用中的调用者还是作为参数：一般而言，如果使用的静态方法或者调用目标明确，那么流内的元素自动作为参数使用；如果函数引用表示实例方法或者不存在调用目标，那么流内的元素自动作为调用目标。
  • 如果一个类中存在同名的实例方法和静态函数，那么编译器就会感到困惑，因为此时编译器并不知道调用哪个方法。(选择静态方法将流内元素作为参数，选择实例方法将流内元素作为调用目标)
  • 方法引用也可以直接使用构造函数

Java8 函数式编程

  • Java8对lambda表达式的处理几乎等同于匿名类的实现，但是在写法上和编程范式上差别较大。
